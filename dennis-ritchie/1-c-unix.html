C and Unix

Two inventions. One language — C — that gave programmers direct power over machines and lived forever. One operating system — Unix — that gave the world a model of computing that everything descended from. And "Hello, World" — the first program every programmer writes, in any language, forever.

// first appeared: Ritchie's 1974 C tutorial
// every programmer's first program, in every language, forever

Part I — The C Language
01

What C gave the world

C (1972) let programmers control memory directly — pointers, structs, manual allocation — while compiling to fast machine code on any hardware. It was the first portable systems language.

before assembly: one program per machine type

after write once, compile anywhere

+1 for the first time, an operating system (Unix) could be written in a high-level language. That changed everything about what was possible.

02

K&R — the canonical book

The C Programming Language (Kernighan and Ritchie, 1978) defined the language so precisely that the first edition became the de facto standard. Its prose is spare, elegant, and still worth reading.

style terse, precise, example-driven

first line introduces Hello, World — immediately

+1 it is the best-selling computer science textbook in history. Generations learned C — and by extension, programming itself — from this book.

03

The C family tree

C became the direct parent of most major programming languages. C++ extended it. Java compiled to a virtual machine patterned on C's model. Python's reference implementation (CPython) is written in C.

children C++ (1985), Objective-C (1984), C# (2000)

grandchildren Java, Python (CPython), Ruby, PHP — all C implementations

+1 JavaScript's engine (V8) is written in C++. The web runs on Ritchie's language family.

04

Direct memory access

C's pointer model — direct access to memory addresses — was controversial and powerful. It made C fast enough for operating systems while still being readable. The tradeoff (power vs. safety) defined systems programming for fifty years.

pointers int *p — the address of a memory location

tradeoff total power, programmer's responsibility for safety

+1 modern "memory-safe" languages (Rust, Swift) exist specifically to solve the problems C's pointers introduced. C defined the problem space.

Hello, World — annotated 
/*
 * "Hello, World" — first appeared in Ritchie's 1974 C tutorial,
 * then in The C Programming Language (Kernighan & Ritchie, 1978, p.6).
 * Every programmer's first program. Every language. Forever.
 */

#include <stdio.h>     /* the standard I/O library — a Ritchie creation */

main()             /* in early C, int was implicit */
{
    printf("Hello, World\n");   /* \n = newline */
}

/*
 * Output: Hello, World
 *
 * This program has been translated into thousands of languages.
 * It is the first thing written. It is written in every language.
 * Ritchie wrote it. Thompson ran it on a PDP-11.
 * It is still running.
 */
Part II — Unix
05

Everything is a file

Unix's deepest idea: files, devices, network connections, and processes all share a single interface. Open, read, write, close. One abstraction for everything. Radical in 1969; standard ever since.

before every device needed its own interface

after one model for disk, keyboard, screen, network socket

+1 this is why Linux can represent a process as a file, a socket as a file, a piece of hardware as a file. The abstraction scales without limit.

06

Composable tools — pipes

Unix's pipe (|) lets the output of one program feed the input of another. Instead of one large program that does everything, you build workflows from small tools that each do one thing well.

philosophy "Do one thing and do it well"

example ls | sort | uniq | wc -l

+1 the pipe operator is the Unix philosophy made concrete. It is the original compositional programming model — the idea that would eventually become functional programming, microservices, and APIs.

07

The portable operating system

The first Unix was written in assembly — locked to one machine. When Ritchie developed C, he and Thompson rewrote Unix in C. For the first time, an operating system could be ported to a new machine by recompiling.

before OS code tied to specific hardware forever

after recompile Unix for a new machine in weeks

+1 this single decision — rewrite the OS in a high-level language — made software portable. Without it, every piece of software today would still be hardware-specific assembly.

08

Multiuser, hierarchical, humane

Unix supported multiple simultaneous users, organized files in a hierarchical directory tree, and had a consistent command-line shell. These were new ideas in 1969. They are now universal assumptions.

shell sh — the command interpreter, another Ritchie/Thompson creation

filesystem / root, directories, permissions

+1 the shell concept — a programmable command language that mediates between user and OS — is now present in every OS on Earth. PowerShell, bash, zsh: all descendants.

The Unix philosophy
"Write programs that do one thing and do it well. Write programs that work together. Write programs to handle text streams, because that is a universal interface." — Doug McIlroy, summarizing the Unix philosophy; Ritchie and Thompson's practice
The Unix family tree
Unix → the operating system beneath every screen on Earth

The work — summary
  • C (1972) — direct memory access, portable compilation, the first language suitable for both systems programming and applications. Still among the two most-used languages in the world.
  • Unix (1969–1973) — everything is a file, composable tools via pipes, the shell, hierarchical filesystem, multiuser, written in C (first OS not written in assembly).
  • The C Programming Language (K&R, 1978) — introduced "Hello, World" to every programmer alive and since. The most influential computer science textbook ever published.
  • Hello, World first appeared in Ritchie's 1974 C tutorial. It is now the first program written by every programmer in every language. An unkillable tradition.
  • The Unix philosophy — do one thing well, compose tools via pipes — became the governing philosophy of software development and is now the model for microservices, APIs, and functional programming.
Confidence and sourcing
  • The history of C and Unix is well-documented in primary sources: Ritchie's own papers ("The Development of the C Language," 1993; "The Evolution of the Unix Time-sharing System," 1984) are available and cited.
  • "Hello, World" appearing first in a 1974 C tutorial is confirmed by Kernighan, who recalls the origin clearly. The first book appearance is K&R 1978, page 6.
  • The family tree above is accurate at a high level. Specific lineage claims (Darwin from BSD, Linux inspired by Unix) are documented and uncontested.
  • The claim that Unix was "the first OS written in a high-level language" is the standard statement. Technically, some earlier experimental systems used high-level languages, but Unix was the first portable, practical, widely-adopted OS written this way.
