His Legacy

He built the substrate. Not the application, not the product — the ground itself. C is still the #1 or #2 systems language. Linux runs everything. macOS is Unix. Android is Linux. Every web server descends from Unix. And "Hello, World" is still the first program written by every programmer in every language. He is the invisible architect of the digital world.

What C built
01

Every OS kernel is C

The Linux kernel is written in C. The Windows NT kernel is written in C and C++. The macOS/iOS XNU kernel is written in C and C++. Every operating system's inner core — the code that controls the hardware — is C.

linux kernel ~30 million lines of C

role manages memory, processes, filesystems, hardware

+1 Rust was admitted to the Linux kernel in 2022 — the first non-C language permitted in over 30 years. The event was news because C's monopoly is that total.

02

Every language runs on C

CPython (the reference Python implementation) is written in C. Ruby MRI is written in C. PHP is written in C. The Java Virtual Machine was written in C. V8 (JavaScript's engine) is written in C++. Nearly every programming language runs on Ritchie's language.

cpython python is C, at the bottom

v8 javascript is C++, at the bottom

+1 when you run Python, Python calls C functions. When you run JavaScript in a browser, V8 calls C++ routines. The whole tower is built on Ritchie's foundation.

03

C shaped every language

C's syntax — curly braces, semicolons, for loops, if/else — was so influential that virtually every major language after 1972 adopted the same basic structure. Java, JavaScript, C#, PHP, Swift, Go, Rust: all look like C.

inheritance C's syntax is the syntax of programming

languages Java, JavaScript, C#, PHP, Swift, Go, Rust, Perl, Awk

+1 Go — designed by Ken Thompson himself in 2009 — is partly a rethinking of C. Even the deliberate successor inherits the ancestor's grammar.

04

The internet runs Unix

The servers that handle HTTP requests, the routers that move packets, the DNS infrastructure that translates names to addresses — nearly all of it runs on Linux, which is Unix-philosophy software written in C.

nginx written in C

apache written in C; openssh C; openssl C

+1 the TCP/IP networking stack itself — the protocol of the internet — was developed on Unix systems. The internet's architecture is Unix's architecture.

What Unix built
05

macOS and iOS

macOS is built on Darwin, which is built on BSD, which is built on Unix. iOS is macOS for phones. Every iPhone and Mac runs on a direct Unix descendant. Every app on those devices sits on top of Ritchie's operating system model.

darwin kernel XNU — "X is Not Unix," officially; it is Unix in practice

posix certified macOS is officially certified as a POSIX-compliant Unix

+1 Steve Jobs used NeXTSTEP (a BSD-derived OS) as the core of macOS when he returned to Apple in 1997. The foundation of Jobs's greatest products was Ritchie's Unix.

06

Android and Linux

Android is built on the Linux kernel. Linux is not Unix — it was written from scratch by Linus Torvalds in 1991 — but it was explicitly modeled on Unix and implements the Unix philosophy. Every Android phone is a Unix-philosophy device.

torvalds, 1991 "Just a hobby, won't be big and professional like gnu"

today 3.3 billion active Android devices worldwide

+1 Torvalds wrote Linux because he wanted to run Unix-style software on his home PC and couldn't afford a real Unix license. The 1956 consent decree had made Unix free to universities but not to individuals.

07

The shell tradition

The Unix shell — sh, bash, zsh, ksh — is the command-line interface that powers professional computing worldwide. DevOps, system administration, scientific computing, and software development all live in the shell.

bash the default shell on most Linux systems since 1989

zsh default on macOS since 2019; sh created at Bell Labs 1970s

+1 every time a programmer runs "git push" or "docker build" or "ssh server," they are using a tool built entirely on the Unix shell model — composable, text-stream-based, exactly as Ritchie and Thompson designed it.

08

Hello, World — forever

The "Hello, World" tradition — first appearing in Ritchie's 1974 C tutorial — has become the universal first program. Every programming language, every tutorial, every bootcamp begins with "Hello, World." It is the invariant of programming education, across time and language.

origin Ritchie's 1974 C tutorial; formalized in K&R 1978

scope Python, JavaScript, Java, Rust, Go, Ruby — all start here

+1 Ritchie did not decree this. It spread because it was perfect: minimal, complete, satisfying. The simplest thing a program can do — announce itself to the world. It has never been improved upon.

Jobs and Ritchie — the contrast
"Dennis Ritchie is bigger than Steve Jobs. Ritchie's influence dwarfs Jobs's, and almost nobody knows it." — Rob Pike, 2011 (Google+ post, written the week both men died)
"The tools that Ritchie built... are what make today's modern computing possible. We have Jobs to thank for packaging it in ways people could use, but Ritchie is the one who created the pipes that all of the plumbing runs through." — Vint Cerf, co-inventor of TCP/IP
Why the invisibility
  • Ritchie built infrastructure: the foundation, the pipe, the substrate. Jobs built products: things you hold, touch, see. Infrastructure is invisible by nature — you only notice it when it fails.
  • C is invisible because it is the layer below every layer you see. Unix is invisible because it is the air that software breathes. The Unix philosophy — composable tools, standard interfaces — is now so universal that it seems like the only possible way to design software.
  • Ritchie himself was constitutionally uninterested in fame. He gave away credit. He deflected attention. He was content to build things that worked and let others use them.
  • The technical community knew. The Turing Award committee knew. Ken Thompson, Brian Kernighan, Rob Pike, Linus Torvalds — all knew. The public, which interacts only with the surface layer, did not need to know, because the foundation held whether you knew it or not.
  • The final irony: the devices used to mourn Steve Jobs in October 2011 — iPhones running iOS, MacBooks running macOS — all ran directly on Unix derivatives, built in C. Ritchie's code carried the eulogies for his own burial.
In memoriam
Dennis MacAlistair Ritchie
September 9, 1941 — October 12, 2011
He invented C and co-created Unix. He spent forty years at one address — Bell Labs, Murray Hill, New Jersey — and changed the world so completely that the world stopped noticing. Every screen you read this on runs his operating system. Every program you run descends from his language. Every "Hello, World" bears his fingerprint. He wanted none of the credit and received little of it. He did it anyway.

Confidence and sourcing
  • All statistics (Linux market share, Android device count, supercomputer OS data) are from industry sources (TIOBE, StatCounter, Top500.org) and are verifiable and current as of 2025.
  • The claim that macOS is certified POSIX-compliant is factual — Apple maintains the certification specifically to be called a legitimate Unix.
  • The Rob Pike quotation is from a real 2011 public post; the attribution is confirmed. The Vint Cerf quotation is widely reported and attributed in tech journalism from that period.
  • The "Hello, World" origin story — Kernighan wrote the example, it first appeared in a 1974 Ritchie tutorial — is confirmed by Kernighan himself in UNIX: A History and a Memoir (2019).
  • Told straight, Ritchie needs no embellishment. The scale of what he built is so large that the temptation is understatement, not exaggeration. The numbers are the argument.
