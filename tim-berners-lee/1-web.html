The Three Inventions

HTML, HTTP, and the URL — three ideas that became the architecture of the modern world

The Three Inventions

When Berners-Lee sat down to build the World Wide Web, he built three things. Not one. Not a single protocol or a single format, but three interlocking pieces that together created something none of them could have done alone. Every website that has ever existed runs on all three. They are unchanged in their fundamentals since 1990.

1

The Format

HTML

HyperText Markup Language. The language for structuring content on a web page. Tags define headings, paragraphs, images, links. The link — the anchor tag — is the key: it connects any document to any other document on the web. HTML is what a browser reads to know what to display.

2

The Protocol

HTTP

HyperText Transfer Protocol. The set of rules for how a browser asks for a document and how a server sends it back. A request goes out — GET / HTTP/1.1. A response comes back — 200 OK — with the document attached. The entire web runs on this conversation, billions of times per second.

3

The Address

URL

Uniform Resource Locator. A unique address for any document anywhere on the internet. http://info.cern.ch/ — the protocol, the server, the path. The URL is what makes it possible to link: you can always say exactly where anything is. Without it, hypertext would have no way to point.

The genius was not in any one of these individually — it was in putting all three together. HTML without HTTP has no way to travel. HTTP without URLs has nothing to address. URLs without HTML have nothing to link. The three form a triangle: each one enables the other two.

The HTTP Request-Response Cycle

Every time you visit a website, the same basic conversation happens. Your browser sends a request to a server. The server finds the document and sends it back. Your browser reads the HTML and displays the page. The entire interaction — for a simple page — takes milliseconds. Berners-Lee defined this conversation in 1990, and it still works the same way.

Animated — HTTP Request / Response

// Step 1 — Browser sends:
GET /index.html HTTP/1.1
Host: info.cern.ch

// Step 2 — Server replies:
HTTP/1.1 200 OK
Content-Type: text/html

// Step 3 — Server sends the document:
<html>
  <head><title>World Wide Web</title></head>
  <body>
    <h1>World Wide Web</h1>
    <p>The WorldWideWeb (W3) is a wide-area hypermedia...</p>
  </body>
</html>

// Step 4 — Browser renders the page. Done.

The Key Insight

Before Berners-Lee, the internet existed. Before Berners-Lee, hypertext existed. Before Berners-Lee, naming systems existed. He did not invent any of the underlying components. What he invented was the specific combination — and the specific decision to make that combination universal and free.

The internet — originally ARPANET — was a packet-switched network that allowed computers to communicate across long distances. By 1989, it was a working global infrastructure connecting universities, research institutions, and governments. But it had no universal way to share formatted documents across it. You could send files, send email, connect to remote terminals — but there was no common standard for a readable, linked document.

Ted Nelson had proposed hypertext in 1965 — the concept of linked documents, where text could contain references (links) to other text. Nelson worked on a system called Xanadu for decades, attempting to build a global hypertext system. It never shipped. The idea was real and powerful; the implementation was never completed.

Berners-Lee's insight was to take Nelson's concept and implement it over the existing internet, using a simple enough system that it could be implemented by anyone, on any computer, in any operating system. The key move was not making it perfect — it was making it simple. Simple enough to implement in an afternoon. Simple enough that anyone could set up a server. Simple enough that the web could grow without central control.

"Any document anywhere can link to any other document anywhere. That is the key idea. Not the technology — the topology." — Paraphrase of Berners-Lee's core insight

01

The Internet (already existed)

TCP/IP, packet switching, global infrastructure connecting computers. ARPANET → Internet. The network was there; it just had no web.

02

Hypertext (already proposed)

Ted Nelson, 1965. The concept of linked documents. Never fully implemented before the web. Berners-Lee took the idea and shipped it.

03

Naming (new contribution)

The URL — a universal address for any document. Without a naming system, hypertext links have nowhere to point. The URL makes the web addressable.

04

The Synthesis

Internet + hypertext + naming = the web. The invention was the combination. All three pieces were necessary. None was sufficient alone.

The First URL and First HTML

The first URL ever used in anger was:

http://info.cern.ch/

Protocol ─────┘ │ │
Server ─────────────┘ │
Path ───────────────────────┘

The first web page, hosted at that address, described the project itself. This was characteristically Berners-Lee: the first thing the web said was an explanation of the web. Documentation before everything else.

The first HTML ever written by Berners-Lee established conventions that persist to this day. The tag structure — opening tag, content, closing tag — is unchanged. The anchor tag with its href attribute — the hyperlink — is unchanged. The basic document structure (head, body, title) is unchanged. The first HTML was also, in its structure, the last HTML: every web page ever written follows its grammar.

<!-- The first web page structure, December 1990 -->
<HEADER>
<TITLE>World Wide Web</TITLE>
<NEXTID> N55</NEXTID>
</HEADER>
<BODY>
<H1>World Wide Web</H1>

The WorldWideWeb (W3) is a wide-area hypermedia
information retrieval initiative aiming to give
universal access to a large universe of documents.

<A HREF="WhatIs.html">What is the World-Wide Web?</A>
</BODY>

<!-- Still accessible: http://info.cern.ch/ -->

That first page is still online, restored by CERN. You can visit it today. It is the oldest active web address in history.

The Decision: No Patent

In April 1993, CERN made a formal declaration that the web technology — the software and the protocols Berners-Lee had developed — was placed in the public domain. Royalty-free. Irrevocably. This was not a default outcome. It was a decision, and Berners-Lee lobbied for it.

The alternative was to patent the web. CERN could have asserted intellectual property rights over HTML, HTTP, and the URL. It could have licensed them — as was common practice with other technologies. It could have collected royalties from every company that built a web server, every developer who used HTML, every browser that implemented HTTP. The accumulated royalties, in the world we actually live in, would have run to hundreds of billions of dollars.

They chose not to. Berners-Lee's argument was clear: the value of the web comes from its universality. A web that costs money to access, or that requires a licence to implement, is not a web — it is a proprietary network, like the many proprietary networks that existed before the web and failed to become universal. The web's power is precisely that everyone, everywhere, can use it identically. Patent protection would have destroyed that.

What Royalty-Free Meant

Berners-Lee has estimated that making the web open rather than proprietary caused it to grow approximately 1,000× faster than a licensed alternative would have. That figure is inherently speculative — there is no counterfactual to measure — but the direction is not in doubt. Openness was the growth engine.

Every company that has ever made money on the web owes that money, indirectly, to this decision. Google. Amazon. Facebook. Netflix. Every web shop. Every news site. Every streaming service. Built on free protocols, openly published, maintained by a standards body Berners-Lee created to keep them free.

The W3C: Keeping the Web Open

In 1994, Berners-Lee left CERN and founded the World Wide Web Consortium (W3C) at MIT, with a European host at INRIA and later additional hosts at Keio University in Japan. The W3C is the international standards body for the web. It develops and maintains the technical specifications that define HTML, CSS, and the other languages and protocols that make the web work.

The W3C's operating principle is that web standards must be royalty-free. Any company that contributes to a W3C standard must agree not to charge royalties for implementing that standard. This is not automatic — the W3C has had to fight for it, and there have been controversies — but it has held. The result is that the specifications for HTML5, CSS, SVG, and every other modern web technology are available for free, for anyone to implement.

Berners-Lee serves as Director of the W3C. His role is partly technical and partly political: ensuring that the web's open character is defended against the constant pressure to fragment it, close it, or monetise access to its core specifications.
